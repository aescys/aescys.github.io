I'll begin here - What is a command line interface, and how is it related to a computers filesystem?

Straight to the point, a command line interface is essentially just a text log that you can use to run commands, which tell your computer what to do. It is used for many, many things - software development, network engineering, version control, video editing, music downloading - if you can think of an operation to be done on a computer there is likely a way to do it with the command line interface. The best introduction to the CLI though, and the one most central to it, is filesystem operations.

Filesystem operations are the everyday things you do with your computers mouse like enter a folder, move, rename, delete, copy a file, open a file. While these all sounds rather plain as described here, doing them through the command line invites a whole new atmosphere. Where before you were observing all from a cool distance, you are now in the meat of the motion along side the computer, feeling more weight to each creation and navigation. Each operating system has it's own command line interface and commands, but these navigational commands are often shared and will be the main topic addressed here.

Where can I find my devices CLI?

On Windows you have two options, the application CMD - Command Prompt, and the application PowerShell. For this tutorial, it is recommended to use Command Prompt

Note: Windows also has the option to use the Windows Subsystem for Linux (WSL) which can create a smoother, less halted feeling environment.

On Linux, your app name may vary depending on your particular distro, but it is likely called Terminal

On macOS, you've got Terminal

As mentioned, in the context of this tutorial the three all have similar commands, but it is worth noting that macOS and Linux share the most similarities. Devices operating on either of these OS's are often referred to in tandem as "unix" devices.

The important part, Commands for your command line interface

Now that you've got the basic understanding of what a command line interface is and where to find it, let's get you familiar with some commands. Try opening your app and running these as you go along.

Unix Devices Windows Usage Example Description
pwd cd pwd <enter> , cd <enter> Shows the path of the directory you are currently working from
ls dir ls <enter> Lists the contents of your active directory
cd cd cd otherDirectory Changes your working folder to the specified one
mv move mv file.txt homework1.txt Renames file.txt to be homework1.txt
mv move mv file.txt projects Moves file.txt into the projects folder
mkdir, md mkdir, md mkdir foo Makes a new folder named foo
rm del rm file.txt Deletes file.txt, with no preservation. CAN NOT be undone. Use caution, don't test your limits.
cp copy cp file.txt projects/ Makes a copy of file.txt inside the projects folder
touch type nul > touch bar.txt , type nul > bar.txt Creates a file of type .txt named bar. This can be used to create an empty file of any extension type.
open start open bar.txt , start bar.txt Launches the selected file with your computers default application to open it.
Note "." ".." "~" and "/" ("\" on windows) all symbolize special locations on your device. Try using them as the location in your cd command.

Some exercises to try out

Excercise One

Commands used: ls, cd, touch(type nul >),mkdir, mv

  1. Open your command line application and use cd and ls (dir for windows) in combination to find your way to your desktop folder.
  2. Create a new folder using mkdir or md. Name it whatever you want
  3. Create a new file using touch or type nul >, also name it however you'd like. Remember, unix vs windows
  4. Move the file into your newly created folder using mv or move
  5. Navigate into the new folder and verify the operation was a success by running ls or dir - if you see your files name indicated in the content list of the folder you have succeeded

Excercise Two

Commands used: touch(type nul >), cp(copy), rm(del), open(start), echo

  1. Navigate to your desktop folder
  2. Create a file called test.txt
  3. Run the command - echo "hello from test" > test.txt
  4. Create a copy of the file on your desktop, naming the file test2.txt
  5. Open the copied file using open or start if you're on windows
  6. Delete (warning) the copied file making sure to provide the correct filename (rm test2.txt, del test2.txt)
  7. Check your trashcan, note that the deleted file is not present there

Bonus: On mac you can use "rm -rf folderName" to delete a folder, and on windows you can use "rmdir /s folderName". Note again, this can be incredibly consequential, do not test these commands out on directories crucial to your devices operating system. Typically important files are anything above the user level directory ("~" on mac)

Don't forget to try running "cd .." or "cd /". These character represent, respectively, the parent directory of the folder you are in and the root of your computer's drive. "." Represents your current folder. It's useful for things like "open .", "start ."".
